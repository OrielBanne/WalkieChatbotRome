## Component Functions

### `render_itinerary(itinerary: Optional[Itinerary])`

Main function to render the complete itinerary display.

**Features:**
- Summary metrics (duration, distance, cost, feasibility)
- Planning notes in expandable section
- Time-ordered list of stops
- Download button for text export

**Usage:**
```python
from src.components.itinerary_display import render_itinerary

# In Streamlit app
render_itinerary(itinerary)
```

### `render_itinerary_map(itinerary: Itinerary)`

Renders an interactive map showing the itinerary route with numbered markers.

**Features:**
- Numbered markers (1, 2, 3...) for each stop
- Route line connecting all stops
- Transport mode colors (blue for pedestrian)
- Detailed popups with stop information
- Interactive zoom and pan

**Popup Content:**
- Stop number and place name
- Visit time and duration
- Place description
- Ticket information and pricing
- Crowd level indicator

**Usage:**
```python
from src.components.itinerary_display import render_itinerary_map

# In Streamlit app
render_itinerary_map(itinerary)
```

### `render_itinerary_stop(stop: ItineraryStop, index: int)`

Renders a single itinerary stop with all details.

**Features:**
- Expandable card for each stop
- Two-column layout (details | ticket/crowd info)
- Place description
- Notes as bullet points
- Ticket information with booking links
- Crowd level indicator

### `render_itinerary_summary(itinerary: Itinerary)`

Displays summary metrics in a 4-column layout.

**Metrics:**
- Total Duration (hours and minutes)
- Walking Distance (kilometers)
- Total Cost (euros)
- Feasibility Score (0-100 with emoji)

### `generate_text_itinerary(itinerary: Itinerary) -> str`

Generates a plain text version of the itinerary for download.

**Sections:**
- Header with generation timestamp
- Summary statistics
- Detailed stop information
- Planning notes
- Footer

### `render_preference_form() -> UserPreferences`

Renders a sidebar form for user preferences.

**Inputs:**
- Interests (checkboxes: art, food, history, photography)
- Hours available (slider: 2-12 hours)
- Maximum budget (number input: 0-500 EUR)
- Max walking distance (slider: 2-20 km)
- Crowd preference (radio: avoid, neutral, don't care)
- Start time (slider: 6:00-14:00)

## Integration

### With Streamlit App

The component is designed to integrate seamlessly with the main Streamlit app:

```python
import streamlit as st
from src.components.itinerary_display import render_itinerary, render_preference_form
from src.planner_integration import plan_itinerary

# Sidebar: Get user preferences
preferences = render_preference_form()

# Main area: Plan button
if st.button("Plan My Day"):
    with st.spinner("Planning your itinerary..."):
        itinerary = plan_itinerary(user_query, preferences)
        
        if itinerary:
            render_itinerary(itinerary)
        else:
            st.error("Failed to generate itinerary")
```

### With Planning Workflow

The component works with the LangGraph-based planning workflow:

```python
from src.agents.workflow import create_planner_workflow
from src.agents.models import PlannerState, UserPreferences
from src.components.itinerary_display import render_itinerary

# Execute workflow
workflow = create_planner_workflow()
state = PlannerState(user_query=query, user_preferences=prefs)
result = workflow.invoke(state)

# Display result
if result.itinerary:
    render_itinerary(result.itinerary)
```

## Data Models

The component uses Pydantic models from `src/agents/models.py`:

### `Itinerary`
- `stops: List[ItineraryStop]` - Time-ordered stops
- `total_duration_minutes: int` - Total trip duration
- `total_distance_km: float` - Total walking distance
- `total_cost: float` - Total cost in EUR
- `feasibility_score: float` - Score 0-100
- `explanation: str` - Planning notes

### `ItineraryStop`
- `time: datetime` - Stop start time
- `place: Place` - Place information
- `duration_minutes: int` - Visit duration
- `notes: List[str]` - Stop-specific notes
- `ticket_info: Optional[TicketInfo]` - Ticket details
- `crowd_level: Optional[CrowdLevel]` - Crowd prediction

### `CrowdLevel` (Enum)
- `LOW` - Low crowds
- `MEDIUM` - Medium crowds
- `HIGH` - High crowds
- `VERY_HIGH` - Very high crowds
